## The semantic contract (agreed, 2026-07-12)

**A Blade program denotes a pure function of its named inputs; compilation is
partial evaluation of that function against whatever prefix of the inputs is
already known.** Files aren't state the program interacts with — they're
arguments the program was applied to. Reading a file at compile time and at
runtime must produce the same value, so *when* the read happens is
unobservable except in cost. Staging is binding-time analysis: inferred,
never meaning-changing.

Three clauses close the model:

1. **Inputs are immutable values within a program's scope.** Fold freely.
   Operational honesty comes from *provenance*, not freshness: record the
   content hash at fold time ("this executable = program · sample.nc@a3f9…").
   Later: memoize compile-time folds by content hash — the compiler becomes
   an incremental build system.
2. **Growing inputs aren't mutable files — they're streams**: arrays with an
   unbounded T-axis. Pure computation over an unbounded T-axis is exactly the
   mergeable CoMoments accumulator (prototype 2); the merge monoid *is* the
   semantics of "the file got longer."
3. **In-place overwrite is out of scope, by design, stated in the spec.**

One declaration, three schedules, chosen by input availability:

| `let a: Dist<F,4> = moments(src, 4)` when… | compiles to |
|---|---|
| src small and closed | folded constant tensor |
| src large and closed | one-pass streaming kernel |
| src open along T | incrementally-merged accumulator |

This is the CompoundIdx pattern (IxKCompound / IxKCompoundDynamic — staged by
input availability, not by declaration) elevated to a language-wide principle.

## The two compiler changes this rests on

1. **`replicate` count = any statically-evaluable int** — LANDED (old
   session): TypeEnv.StaticValues populated in checkModule's pre-pass by the
   same StaticEval.resolveStatics lowering runs; inferReplicate normalizes
   the count to a literal in the typed tree; corpus replicate/005. This is
   the "static k drives elaboration, A rides through" staging that
   `moments(A, k)` needs.
2. **`let static` = assertion (fold or fail loudly)** — bare `let` means
   "stage as you see fit" (the compiler may fold or not, a cost decision);
   `let static` *demands* a compile-time value, and failure to fold is a
   compile error, not today's silent demotion to runtime. Exemption: a
   lambda-valued static declares a function (the marker means immutability
   there), not a foldable value. Landing alongside: module-level tuple
   destructuring actually folds (`let static (a, b) = pair()` leaves were
   skipped by resolveStatics; only a runtime projection fallback made it
   work), so destructured components can drive elaboration positions like
   the replicate count.
   Known gap, deliberate: cross-module static value imports aren't visible
   to typecheck's per-module resolveStatics (no corpus coverage today);
   fix belongs with provider-backed statics.
